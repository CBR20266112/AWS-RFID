RFID Access Control & Statistics - Developer Guide

RFID 리더기(ESP8266 + PN532), 웹 대시보드, PHP API 및 MySQL(MariaDB) 데이터베이스 전체 연동 가이드

1. 시스템 구성도 (Architecture Overview)

본 시스템은 하드웨어 단말기에서 카드를 센싱하여 서버에 기록하고, 관리 웹 브라우저를 통해 통계 및 사용자 관리를 지원하는 풀스택 IoT 시스템입니다.

동작 흐름:
RFID 카드 접촉 ➔ ESP8266(NTP 시간 적용) ➔ AWS API(HTTP POST) ➔ MySQL(MariaDB) 적재 ➔ 실시간 대시보드 반영(Charts.js 시각화 & 팝업 등록 폼)

2. 하드웨어 배선 및 핀맵 (Hardware Wiring)

NodeMCU V2 (ESP8266) 개발 보드와 각 센서 모듈의 연결 배치입니다. PN532 리더기는 I2C 통신 모드로 설정되어 있어야 합니다.

센서/모듈 센서 핀 NodeMCU 핀 비고
PN532 RFID SDA D5 (GPIO14) I2C 데이터 라인 (공유)
PN532 RFID SCL D6 (GPIO12) I2C 클럭 라인 (공유)
OLED 디스플레이 SDA D5 (GPIO14) I2C 데이터 라인 (공유)
OLED 디스플레이 SCL D6 (GPIO12) I2C 클럭 라인 (공유)
패시브 부저 (+) 핀 D7 (GPIO13) 멜로디 및 경고음 출력 (PWM)
온보드 LED - D4 (GPIO2) 보드 상태 인디케이터 (Active LOW)

3. 데이터베이스 설계 (Database Schema)

AWS Lightsail 서버 내 로컬 MariaDB에 생성된 스키마 구조입니다. 데이터베이스명은 rfid_system입니다.

사용자 테이블 (users)

CREATE TABLE users (
    uid VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    department VARCHAR(100),
    phone VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

출입 기록 테이블 (access_logs)

CREATE TABLE access_logs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    uid VARCHAR(50) NOT NULL,
    name VARCHAR(100) DEFAULT 'Unregistered',
    status VARCHAR(20) NOT NULL, -- 'granted' or 'denied'
    accessed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

4. 백엔드 API 명세 (`api.php`)

백엔드 API는 PHP 8.5로 작성되어 있으며 웹 루트 디렉토리(/var/www/html/api.php)에 위치합니다. 주요 액션 쿼리 파라미터는 다음과 같습니다.

1. 카드 감지 기록 (`POST /api.php?action=scan`)

ESP8266 단말기가 카드를 읽었을 때 서버에 호출하는 API입니다.

요청 바디 (JSON):

{
    "uid": "49 6B 96 29"
}

응답 바디 (JSON):

{
    "status": "granted", -- 또는 "denied"
    "name": "홍길동",     -- 미등록시 "Unregistered"
    "uid": "49 6B 96 29",
    "timestamp": "2026-06-24 17:05:00"
}

2. 신규 사용자 등록 (`POST /api.php?action=register`)

대시보드에서 등록되지 않은 사용자의 카드 정보를 입력할 때 호출하는 API입니다.

{
    "uid": "8A F9 D2 E3",
    "name": "강감찬",
    "department": "기술지원팀",
    "phone": "010-9876-5432"
}

3. 접속 기록 CSV 다운로드 (`GET /api.php?action=download_csv`)

엑셀 호환성(UTF-8 BOM 포함)을 고려해 데이터베이스의 전체 출입 기록을 CSV 파일로 반환합니다.

5. 대시보드 주요 특징 (`index.html`)

  • 차트 시각화: Chart.js를 연동하여 '최근 7일간 출입량', '24시간대별 출입량 분포', '출입 승인/거부 비중' 총 3개의 반응형 통계 그래픽을 렌더링합니다.
  • 자동 팝업 등록 기능: 1.5초 주기 실시간 폴링 중 미등록 카드 태깅이 인식되면 화면에 즉시 신규 등록 모달(Modal) 창이 발생하여 편의성을 높였습니다.
  • 테마 모드: CSS Variables 구조로 구성되어 다크 모드(디폴트)와 라이트 모드 테마 간 전환이 가능합니다.

6. 하드웨어 빌드 및 배포 방법 (Firmware Deployment)

Arduino CLI 명령어를 사용해 코드를 빌드하고 보드에 배포합니다. 환경변수 로드가 필요합니다.

컴파일 (Compile)

arduino-cli compile --fqbn esp8266:esp8266:nodemcuv2 c:\Users\SW중심대학사업다\.antigravity-ide\esp8266_oled

업로드 (Upload)

arduino-cli upload -p COM3 --fqbn esp8266:esp8266:nodemcuv2 c:\Users\SW중심대학사업다\.antigravity-ide\esp8266_oled
